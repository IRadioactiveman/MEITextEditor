Graphic MEI Generator Help

Welcome to the Graphic MEI Generator! This tool was designed to make it easier to create XML files containing neumes according to the MEI schema. Please visit the official Music Encoding Initiative website for more information about MEI.

Important! The tool is designed as a single page application, which means that it is not advised to use the refresh or the back button. If you do all your progress will probably be lost and you have to start all over again! If you want to change any mistakes you made, please do so by hand or with the opportunities given on the site.

UI

Zhe user interface is dividedin three sections, on the left is an input mask where you enter all the data, right next to it is the xml output, which will change by your input. Below those two is a raw graphical output of the syllables and pitches you added.

General workflow

The process is divided into a couple of steps:

Meta data:
This section contains general data about the work itself as well as information about the file itself.

Source data:
This section contains information about the sources of the musical work. Multiple sources may be added, at least one source is mandatory.

Staff data:
This section contains information about the staffs, e.g. the lines the notes are painted on, including associated clefs.

Syllable data:
This section displays the muscial content itself. First, there is the words that are sung, then you may enter the sung pitches, including variations between sources. To make input easier, the basic neumes virga, punctum, pes, clivis, torculus, porrectus, climacus and scandicus are pre programmed, so don't worry if you add a base pitch and the rest will be generated while you are redirected to the syllable form.

Correcting errors

From time to time an error will occur. If you are in the middle of creating the document, you can use one of the "change value" buttons on the top or the bottom of the site. You will be able to select the source, staff, syllable or any other element you created, and the on the right side you will see the current values of the selected item. You can then change the values, create new ones or delete values. When you are finished there you can go back to the respective form. Watch out: the pre programmed neume types will not be considered if you use an "insert syllable" button! They will have to be inserted by hand.

Graphical output

The graphical output is for the unexperienced xml user to control his inputs concerning syllables and pitches. It is by no means 100% accurate, but it gives you the idea of what you are doing. First of all, there is only one octave displayed, so don't worry if you have two c's in two octaves and they are displayed on the same line. Red notes mean there is no given pitch, and blue notes are from variations.

This tool does not validate your data against the MEI schema since proper Relax NG validators for javascript were not found. The tool has been properly tested, but to be sure you should consider validating the resulting MEI file yourself. I had good experiences with Jing, an easy to use Java command line tool for validating Relax NG schemata amongst others.
